## 怎麼新增一場 webinar（給未來的自己）

### 1. 開資料夾

照 `NN-kebab-slug/` 編號連續開：

```
04-<下一場主題的 kebab slug>/
```

### 2. 必備檔案

每個資料夾結構固定六檔：

| 檔名 | 內容 | 是否 commit |
|---|---|---|
| `video.mp4` | 原片 | ❌ gitignored（檔案太大留本地） |
| `transcript.txt` | 純文字逐字稿 | ✅ |
| `transcript.srt` | 帶 timestamp 字幕 | ✅ |
| `transcript.vtt` | WebVTT 字幕 | ✅ |
| `transcript.json` | qwen-asr 原始輸出 | ✅ |
| `notes.md` | take-note skill 產出 | ✅ |

下載+轉錄走 [[yt-dlp]] → [[qwen-asr]] pipeline，產出已含前 4 個。`notes.md` 走 [[take-note]] skill 從 `transcript.txt` 生。

### 3. 注釋 MOC

寫完 notes.md 之後，**回到本檔做四件事**：

1. 「全部 webinars」表加一列
2. 「按主題找」標籤頁掃過 — 新場碰到的概念有對應 tag 就加 link，沒有就新開
3. 如有新類型框架 / 警告 / 案例，補進對應 section
4. 跨場觀察累積到值得更新時，動 README.md（不要每場都動）

### 4. Commit message 風格

照 conventional commits：

```
feat(04-<slug>): add <title> notes + transcripts
```

或新場分兩 commit：

```
chore(04-<slug>): add transcripts (txt/srt/vtt/json)
feat(04-<slug>): add take-note knowledge map
```

- **想找某個概念在哪場出現** → 看「按主題找」
- **想知道某場在講什麼** → 點對應 notes.md
- **想看 framework 跨場對照** → README.md
- **想下載原片** → mp4 在本地 `~/Documents/gartner-ai-webinars/NN-*/video.mp4`，沒上 git
- **想自己重做 notes** → 跑 `/take-note`，源指向 `transcript.txt`
